Power BI CI/CD Architecture

GitHub Actions + Windows self-hosted runner + Tabular Editor 2 + Power BI REST/XMLA deployment flow

Deployment Flow

  • • Push to main or run workflow manually
  • • Detect changed project folders from Git history
  • • Deploy each selected project through matrix jobs
  • • BIM deploy runs before PBIX report publish

Security Boundary

  • • Secrets stay in GitHub Actions settings
  • • Service principal authenticates with Entra ID
  • • Runner uses outbound HTTPS connections
  • • No inbound GitHub connection to the VM is required

Power BI Integration

  • • Tabular Editor 2 publishes BIM through XMLA
  • • Python publishes PBIX through REST imports
  • • Existing reports are updated idempotently
  • • Reports are rebound to same-name semantic models
